# 嵌入形式

## 核心理念

判镜不应该是独立 App。它嵌入在用户已经在用的 LLM 使用过程中。

四种嵌入形式，按侵入性从低到高排列。

---

## 形式一：System Prompt 注入

**侵入性：零。安装方式：粘贴文本。**

判镜的 MVP 就是一段 300 字的 system prompt。用户贴在 Claude 的 Project Instructions、ChatGPT 的 Custom Instructions、或 Claude Code 的 CLAUDE.md 里，LLM 自己变成判镜。

**System Prompt 核心逻辑：**

```
当检测到用户在做决策时：
1. STOP——先问："这个决策，你希望事后怎么判断它对错？"
2. DIMENSIONS——拆 3-5 个关键维度，0-5 打分
3. HISTORY——同类决策历史对照
4. VERDICT——综合判断 + 置信度标注
5. LOG——记录决策点，下次复盘时调出对照
```

**优势：** 零安装、零成本、立刻可用。作为冷启动验证极佳。
**局限：** 无持久存储（依赖对话历史）、无跨会话记忆、无推送能力。

---

## 形式二：MCP Server

**侵入性：极低。安装方式：拖入 settings.json 一个配置。**

Claude Code 已有 MCP 协议。判镜作为 MCP Server 挂在 Claude 背后。

用户正常聊天。Claude 检测到"决策时刻"，自动调用判镜工具：

```
内部调用：
→ capture_decision(domain, context)
→ recall_context(decision_id)
→ recalibrate(domain)

用户看到的结果跟正常聊天一样，
底部多一行小字: 📋 已记录 · 决策#47
```

MCP Server 做的事：
- `capture_decision` — 捕获决策点，结构化存储
- `recall_context` — 自动调出历史相关判断
- `recalibrate` — 对照数据修正权重
- `push_reminder` — 数据回收期到，主动推送

**优势：** 有持久存储、模型主动调用、用户几乎无感。
**适用：** Claude Code 深度用户。

---

## 形式三：浏览器插件

**侵入性：低。安装方式：浏览器扩展商店一键安装。**

覆盖所有 Web 端 LLM（ChatGPT/Claude.ai/DeepSeek 网页版）。

在网页右侧加窄栏，默认折叠。当对话触发"判断信号"时展开。

**触发信号（本地识别，不传服务器）：**
- "该不该""要不要""选哪个""帮我决定"
- "这个能做吗""哪个方案好"
- 用户手动点击插件图标

**展开内容：** 检测到的决策类型、历史同类决策、快速维度拆解入口、个人判断偏好摘要。

**跟 MCP 的区别：** MCP 是模型主动调用工具。插件是人主动查看自己的判断空间。前者"模型替你记"，后者"你随时能看到全景"。

---

## 形式四：API 中间层

**侵入性：中。安装方式：修改 API endpoint。**

面向开发者/团队。用户继续用自己喜欢的 LLM API，请求链路加判镜中间层：

```
用户代码 → 判镜中间层 → LLM API
               ↓
         判断数据库
```

中间层做的事：
- 拦截请求，识别"判断类对话"
- 注入历史同类决策 + 当前 rubrics 权重
- 发给 LLM 的 prompt 被增强了
- LLM 返回的响应中，提取决策点自动存入判断数据库
- 定期检查"需要回收数据的决策"

**用户感知不到判镜。** 但 LLM 突然变得更"懂他们"。

**适用：** 团队/企业。最高价值，最难替换。

---

## 四种形式的递进路径

```
System Prompt 注入
    零安装、零成本、零持久化
    验证需求、积累早期用户
          ↓ 验证通过
MCP Server
    有持久存储、模型主动调用
    Claude Code 用户专属
          ↓
浏览器插件
    跨平台、有 UI、被动触发
    覆盖所有 Web 端 LLM 用户
          ↓
API 中间层
    团队/企业级、全链路嵌入
    最高价值、最难替换、B2B
```

---

## 为什么不一开始就做大产品

每一步的研发成本递增，但每一步的验证信号都是上一步给的。

先验证"用户真的需要判断助手吗"（system prompt 的下载数 + 反馈）。
再验证"用户愿意让 AI 持久记录决策吗"（MCP Server 的安装数 + 留存）。
再验证"用户愿意在非 Claude Code 环境用吗"（插件的安装数）。
最后验证"团队愿不愿意付费"（API 中间层的 B2B 线索）。

**不需要一开始就做大产品。** 一段 300 字的 system prompt 就是 MVP。
