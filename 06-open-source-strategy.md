# 开源策略

## 核心判断

**判镜的核心资产不是代码——是 rubrics 权重和校准数据。**

代码开源不丢护城河。品类定义权才是护城河。

---

## 开源的好处

### 1. 品类定义加速器

不开源：你一个人说服市场"这个东西存在"。
开源：几百个用户帮你定义"判断工程化"是什么。

对于品类定义者，速度比控制重要。先成为默认标准，后来者叫"判镜的替代品"——名字是你定的。

### 2. 校准数据飞轮的启动方式

判镜的长期价值来自领域私有校准数据。但一个人做不出多领域的数据。

需要 100 个创作者 + 50 个产品经理 + 30 个创业者用判镜，他们的 rubrics 和校准历史（脱敏后）就是数据飞轮。

开源协议约定：**个人数据归你，聚合后的领域校准权重改善归项目。**

用户没损失。你得到一个不断进化的 rubrics 引擎。

### 3. 信任的默认状态

判镜参与用户的**判断过程**——决策历史、判断偏好、甚至价值观。闭源 SaaS 获取这种数据的信任成本极高。

"你是不是在用我的决策数据训练模型？"
"你是不是在反向影响我的判断？"

开源回答了一切。代码公开、存储方式公开、权重算法公开。用户可以跑在自己的机器上，数据不出本地。

**对"判断"这种高敏感度场景，开源不是道德选择，是商业选择。闭源反而更难卖。**

### 4. Agent 生态入口

MCP 生态正在形成。做 MCP Server 的少，每个细分方向只有一两个玩家。

判镜是"判断工程化"方向唯一的 MCP 实现。当 MCP 市场/目录出现时，判镜就是"决策助手"类 MCP 的默认推荐。

**这个位置现在是空的。谁先占住就是谁的。**

## 开源的收益

### 短期（0-6 个月）

| 收益 | 预期 |
|------|------|
| GitHub Stars | 2000-5000（如果定位准 + 有内容配合） |
| 早期用户池 | 200-500 实际使用者，50-100 活跃贡献校准数据 |
| 内容素材 | 用户 use case、对比截图、准确率提升数据 |
| 品类占位 | "判镜=判断工程化"在搜索引擎和开发者心智中的等同关系 |

### 中期（6-18 个月）

| 收益 | 预期 |
|------|------|
| 校准数据资产 | 按领域拆开的 rubrics 权重模板（创作者版、PM版、创业者版……） |
| 招聘/合作信号 | 贡献 PR 的人是天然早期员工。一次 PR 比十次面试准 |
| 企业线索 | 用了开源版的团队 leader，"想部署私有版，能不能付费" |
| 品牌红利 | 中文 AI 开源生态"判断工程化"赛道定义者 |

### 长期（18 个月+）

| 收益 | 预期 |
|------|------|
| 企业版（闭源） | 团队协作、私有部署、SAML SSO、审计日志 |
| 校准权重 marketplace | 社区产出的领域模板，运营一个交易市场 |
| 培训/认证 | "判镜认证的判断工程师" |

## 开源的风险

| 风险 | 严重程度 | 应对 |
|------|----------|------|
| 大厂复制 | 中 | 大厂复制代码不复制校准数据和品类定义。代码不值钱，品类认知和社区值钱 |
| 没人用 | **高** | 真正的风险。冷启动比代码质量重要。需要前 10 个种子用户——自己、认识的内容创作者、AI 圈 KOL |
| 社区分裂 | 低 | 行业太早期，现在没社区，不存在分裂 |
| 维护负担 | 中 | MCP Server 代码量很小（几百行）。核心引擎才是重的，不管开不开源都要迭代 |

---

## 推荐策略

**核心引擎开源。垂直场景模板部分免费部分付费。企业协作版闭源。**

- 开源 = 品类定义权 + 数据飞轮 + 信任 + 生态入口
- 模板付费 = 创作者版 rubrics 免费，高级定制版付费
- 企业闭源 = 团队协作 + SaaS 月费 + 私有部署

---

## 跟 GitHub/GitLab 的类比

- GitHub 开源了 git 的协作层，企业版（GitHub Enterprise）闭源
- GitLab 开源了 CE（社区版），EE（企业版）闭源

判镜同理：**个人用判镜应该是免费的、开源的。**
**团队/企业用的协作+私有部署功能，可以闭源收费。**

开源和商业化不矛盾。开源是商业化的加速器，不是替代物。
